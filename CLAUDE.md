# CLAUDE.md — Instruções do projeto

## TAREFA FIXA (obrigatória no início de cada sessão)

Sempre que abrir o terminal / iniciar uma sessão nesta pasta:

1. **Ler `context.md`** — para entender o que é o projeto, a arquitetura e as regras.
2. **Ler `atualizacoes.md`** — para saber o que já foi feito e onde paramos.

## Regra permanente ao trabalhar aqui

- **A cada mudança** feita no projeto (código, planilha, backend Apps Script, config, etc.),
  registrar uma nova entrada **no topo** de `atualizacoes.md` (data, arquivos, o que mudou, motivo, status).
- Se algo estrutural mudar (arquitetura, listas fixas, novas abas, nova ação da API),
  atualizar também o `context.md`.

## Idioma

- Responder e documentar em **português (Brasil)**.
